# CSS Units

---

# Why CSS Units Exist

Every visual property in CSS needs a value.

Examples:

```css
width: 300px;
font-size: 2rem;
margin: 5%;
height: 50vh;
```

The question is:

> 300 WHAT?
>
> 2 times WHAT?
>
> 50% OF WHAT?

CSS units answer that question.

They define **how measurements are calculated**.

Without units, the browser cannot know how large something should be.

---

```
CSS Units

├── Absolute Units

    ├── vmax

```

# 1. Absolute Units

## What Are They?

Absolute units have fixed sizes.

Their values do not depend on parents, viewport size, or font size.

---

## px (Pixels)

### What is it?

The most commonly used CSS unit.

```css
width: 300px;
font-size: 16px;
```

---

### How It Works

Historically:

```
1 px = 1 physical screen pixel
```

Modern browsers use:

```
CSS Pixel ≠ Device Pixel
```

Browsers scale CSS pixels according to device density.

Example:

An iPhone may use:

```
1 CSS px = 3 device pixels
```

This allows websites to appear similarly sized across devices.

---

### Why It Exists

To provide predictable sizing.

---

### Advantages

```text
✓ Easy to understand
✓ Precise control
✓ Consistent appearance
✓ Most widely used
```

---

### Disadvantages

```text
✗ Not flexible
✗ Doesn't adapt to user font preferences
✗ Can reduce accessibility if overused
```

---

### Use When

```text
• Borders
• Small spacing
• Icons
• Shadows
• Precise positioning
```

---

### Avoid When

```text
• Main typography
• Responsive layouts
```

---

# Other Absolute Units

---

## cm

Centimeters.

```css
width: 5cm;
```

Mostly intended for print.

---

## mm

Millimeters.

```css
width: 20mm;
```

Rarely used.

---

## in

Inches.

```css
width: 2in;
```

Mostly print-related.

---

## pt

Points.

```
1pt = 1/72 inch
```

Used in typography and print.

---

## pc

Picas.

```
1pc = 12pt
```

Very uncommon.

---

# Why Web Developers Rarely Use These

Browsers display content on screens with varying densities.

Physical measurements become unreliable.

Therefore:

> px dominates screen design.

---

# 2. Relative Units

Relative units calculate size based on another reference.

They adapt automatically.

This is the foundation of responsive design.

---

# Percentage (%)

## What Is It?

A percentage of another value.

```css
width: 50%;
```

---

## Relative To What?

Depends on the property.

Examples:

### Width

Relative to parent width.

```css
.parent{
    width:400px;
}

.child{
    width:50%;
}
```

Result:

```
Child width = 200px
```

---

### Height

Only works reliably if parent height exists.

```css
height:50%;
```

---

### Margin/Padding

Percentage is based on parent width.

Even vertical padding.

Example:

```css
padding-top:10%;
```

Uses parent width.

This surprises many developers.

---

### Advantages

```text
✓ Responsive
✓ Flexible
✓ Parent-based layouts
```

---

### Disadvantages

```text
✗ Depends heavily on context
✗ Can be confusing
```

---

### Use When

```text
• Widths
• Responsive containers
```

---

# em

## What Is It?

Relative to the font-size of the element.

---

Example

```css
font-size:20px;

padding:2em;
```

Calculation:

```
2 × 20px

= 40px
```

---

# Internal Logic

```
em
↓
Uses current element font-size
↓
Calculates measurement
```

---

Example

```css
.parent{
    font-size:20px;
}

.child{
    font-size:2em;
}
```

Result:

```
Child font-size

= 2 × 20

= 40px
```

---

# The Compounding Problem

Example:

```css
.parent{
    font-size:20px;
}

.child{
    font-size:2em;
}

.grandchild{
    font-size:2em;
}
```

Calculation:

```
Parent      = 20px

Child       = 40px

Grandchild  = 80px
```

Each level multiplies again.

---

# Advantages

```text
✓ Scales naturally
✓ Useful for reusable components
```

---

# Disadvantages

```text
✗ Nested scaling becomes unpredictable
✗ Hard to debug
```

---

# Use When

```text
• Component spacing
• Buttons
• Self-contained widgets
```

---

# Avoid When

```text
• Deep nesting exists
```

---

# rem

## What Is It?

Root em.

Relative to:

```html
<html>
```

font-size.

---

Example

Default browser:

```css
html{
    font-size:16px;
}
```

Then:

```css
font-size:2rem;
```

Calculation:

```
2 × 16

= 32px
```

---

# Why rem Exists

Problem with em:

```
Nested multiplication.
```

Solution:

```
Always reference root.
```

---

# Internal Logic

```
rem
↓
Find html font-size
↓
Multiply
↓
Done
```

---

Example

```css
html{
    font-size:16px;
}

.parent{
    font-size:50px;
}

.child{
    font-size:2rem;
}
```

Result:

```
Child = 32px

NOT 100px
```

---

# Advantages

```text
✓ Predictable
✓ Accessible
✓ Responsive typography
✓ Industry standard
```

---

# Use When

```text
• Font sizes
• Margins
• Padding
• Layout spacing
```

---

# Industry Practice

Many teams use:

```css
html{
    font-size:16px;
}
```

Then:

```css
1rem = 16px
2rem = 32px
0.5rem = 8px
```

---

# vw

## What Is It?

Viewport Width.

```
1vw = 1% of viewport width
```

---

Example

Browser width:

```
1200px
```

Then:

```css
width:50vw;
```

Calculation:

```
50% of 1200

= 600px
```

---

# Advantages

```text
✓ Extremely responsive
✓ Perfect for hero sections
```

---

# Use When

```text
• Full-width layouts
• Responsive typography
```

---

# Avoid When

```text
• Small text
• Critical UI controls
```

---

# vh

## What Is It?

Viewport Height.

```
1vh = 1% viewport height
```

---

Example

Browser height:

```
800px
```

Then:

```css
height:100vh;
```

Result:

```
800px
```

---

# Common Usage

Full-screen sections.

```css
.hero{
    height:100vh;
}
```

---

# Mobile Issue

Mobile browsers hide/show address bars.

This changes viewport height.

Result:

```text
100vh can jump unexpectedly.
```

Modern alternatives:

```css
100dvh
100svh
100lvh
```

---

# vmin

## What Is It?

Uses the smaller viewport dimension.

Formula:

```
vmin = min(vw, vh)
```

---

Example

Viewport:

```
1200 × 800
```

Then:

```
1vmin = 8px
```

Because:

```
800 is smaller.
```

---

# Use When

```text
• Maintaining proportions
• Responsive squares
```

---

# vmax

## What Is It?

Uses the larger viewport dimension.

Formula:

```
vmax = max(vw, vh)
```

---

Example

Viewport:

```
1200 × 800
```

Then:

```
1vmax = 12px
```

---

# Use When

```text
• Creative layouts
• Dynamic scaling
```

---

# ch

## What Is It?

Width of the "0" character.

Example:

```css
width:30ch;
```

Means:

```
About 30 characters wide.
```

---

# Why It Exists

Readable text.

---

# Use When

```text
• Form fields
• Articles
• Limiting line length
```

---

Example

```css
article{
    max-width:65ch;
}
```

Improves readability.

---

# ex

## What Is It?

Height of lowercase "x".

Depends on font.

---

# Why Rarely Used

```text
✗ Inconsistent
✗ Font-dependent
✗ Unpredictable
```

---

# Unit Comparison

| Unit | Relative To | Responsive | Common Usage |
|---------|------------|-----------|-------------|
| px | CSS Pixel | No | Borders, icons |
| % | Parent | Yes | Widths |
| em | Current font-size | Yes | Component scaling |
| rem | Root font-size | Yes | Typography |
| vw | Viewport width | Yes | Full-width layouts |
| vh | Viewport height | Yes | Hero sections |
| vmin | Smaller viewport side | Yes | Proportional sizing |
| vmax | Larger viewport side | Yes | Dynamic scaling |
| ch | Character width | Yes | Text readability |
| ex | x-height | Limited | Rare |

---

# How Professionals Usually Choose Units

Typography:

```css
font-size: rem;
```

Spacing:

```css
margin: rem;
padding: rem;
```

Borders:

```css
border: px;
```

Responsive Containers:

```css
width: %;
```

Hero Sections:

```css
height: vh;
```

Readable Articles:

```css
max-width: ch;
```

# Interview Questions

## Difference between em and rem?

```text
em  → Relative to current element font-size.

rem → Relative to root html font-size.
```

---

## Why is rem preferred?

```text
Predictable scaling.

Avoids em compounding.
```

---

## What is 1vw?

```text
1% of viewport width.
```

---

## What is StopIteration equivalent here?

```text
Not applicable.

Viewport and font units do not iterate;
they calculate measurements dynamically.
```

# Mental Model

Think of CSS units as answering this question:

```
"Compared to what?"
```

Absolute units say:

```
Use this exact size.
```

Relative units say:

```
Calculate my size from something else.
```

Mastering CSS units means understanding those reference points.

Once you know what each unit measures itself against, responsive CSS stops feeling magical and starts feeling predictable.
