# HTML Forms — Understanding-First Notes

---

# What is a Form?

An HTML form is a container used to collect data from users and send that data to a server for processing.

Examples:

- Login pages
- Registration pages
- Search boxes
- Contact forms
- Checkout pages
- Surveys
- Feedback forms

Without forms, websites would only display information.

Forms make websites interactive.

---

# Why Do Forms Exist?

Imagine a website without forms.

Users could:

- Read content
- View images
- Click links

But they could NOT:

- Log in
- Create accounts
- Search products
- Submit feedback
- Upload files
- Make purchases

Forms solve this problem.

They allow users to provide input.

---

# Real World Flow

```
User
  ↓
Fills Form
  ↓
Browser Collects Data
  ↓
Form Submitted
  ↓
Server Receives Data
  ↓
Server Processes Data
  ↓
Response Sent Back
  ↓
User Sees Result
```

Example:

```
Username: John
Password: 1234

↓ Submit

Browser sends:

username=John
password=1234

↓

Server checks credentials.

↓

Login Success / Failure
```

# Important Form Attributes

---

## action

### What is it?

Specifies where the form data should be sent.

Example:

```html
<form action="/login">
</form>
```

---

### Internal Working

User clicks Submit.

Browser packages data.

Browser sends it to:

```
/login
```

---

### Example

```html
<form action="/register">
</form>
```

Data goes to:

```
example.com/register
```

---

## method

### What is it?

Determines HOW data is sent.

Two common methods:

- GET
- POST

---

# GET Method

## Syntax

```html
<form method="GET">
```

---

## Internal Working

Data is appended to URL.

Example:

Input:

```
search = laptop
```

URL becomes:

```
example.com?search=laptop
```

---

## Use GET When

✓ Searching

✓ Filtering

✓ Bookmarkable pages

✓ Non-sensitive data

---

## Avoid GET When

✗ Passwords

✗ Credit card details

✗ Large data

---

# POST Method

## Syntax

```html
<form method="POST">
```

---

## Internal Working

Data is sent inside request body.

URL stays clean.

Example:

```
POST /login

username=john
password=123
```

---

## Use POST When

✓ Login

✓ Registration

✓ Payments

✓ Sensitive data

✓ File uploads

---

## GET vs POST

| Feature | GET | POST |
|----------|-----|-------|
| Visible in URL | Yes | No |
| Bookmarkable | Yes | No |
| More Secure | No | Better |
| File Upload | No | Yes |
| Search Forms | Yes | Sometimes |
| Login Forms | No | Yes |

# Input Elements

---

# `<input>`

## What is it?

Most commonly used form control.

Allows users to enter data.

---

## Syntax

```html
<input type="text">
```

---

# Text Input

```html
<input type="text">
```

Used for:

- Name
- City
- Username

Example:

```html
<label>Name:</label>
<input type="text">
```

---

# Password Input

```html
<input type="password">
```

Displays:

```
*******
```

instead of actual characters.

Used for:

- Passwords
- PINs

---

# Email Input

```html
<input type="email">
```

Purpose:

Collect email addresses.

Browser performs validation.

Example:

```html
<input type="email">
```

Valid:

```
abc@gmail.com
```

Invalid:

```
abcgmail
```

---

# Number Input

```html
<input type="number">
```

Used for:

- Age
- Quantity
- Price

Example:

```html
<input type="number" min="1" max="100">
```

---

# Date Input

```html
<input type="date">
```

Provides date picker.

Used for:

- DOB
- Appointments

---

# Radio Buttons

## Purpose

Choose ONE option.

Example:

```html
<input type="radio" name="gender"> Male

<input type="radio" name="gender"> Female
```

---

## Why same name?

Because radio buttons sharing the same name form one group.

Only one can be selected.

---

# Checkboxes

## Purpose

Choose MULTIPLE options.

Example:

```html
<input type="checkbox"> Cricket

<input type="checkbox"> Football

<input type="checkbox"> Music
```

User can select:

```
✓ Cricket
✓ Music
```

simultaneously.

---

# File Upload

```html
<input type="file">
```

Used for:

- Resume uploads
- Profile pictures
- Documents

---

# Hidden Input

```html
<input type="hidden">
```

Invisible to users.

Stores extra information.

Example:

```html
<input type="hidden" name="userid" value="101">
```

---

# Submit Button

## Purpose

Submits the form.

Example:

```html
<input type="submit">
```

or

```html
<button type="submit">
    Submit
</button>
```

---

# Reset Button

## Purpose

Clears form values.

Example:

```html
<input type="reset">
```

---

# Labels

## What are Labels?

Describe what an input field means.

Example:

```html
<label>Email</label>

<input type="email">
```

---

## Better Approach

```html
<label for="email">Email</label>

<input id="email" type="email">
```

---

## Why?

Clicking label focuses the input.

Improves accessibility.

---

# Textarea

## Purpose

Multi-line text input.

Example:

```html
<textarea rows="5" cols="30"></textarea>
```

Used for:

- Comments
- Feedback
- Messages

---

# Select Dropdown

## Purpose

Choose one option from many.

Example:

```html
<select>
    <option>India</option>
    <option>USA</option>
    <option>Japan</option>
</select>
```

---

# Multiple Select

```html
<select multiple>
    <option>Java</option>
    <option>Python</option>
    <option>JavaScript</option>
</select>
```

Allows multiple choices.

---

# Placeholder

## Purpose

Shows hint text.

Example:

```html
<input
    type="text"
    placeholder="Enter username">
```

Displays:

```
Enter username
```

until user types.

---

# Required Attribute

## Purpose

Prevents empty submission.

Example:

```html
<input
    type="email"
    required>
```

Browser validation occurs automatically.

---

# readonly

## Purpose

User can view but cannot edit.

Example:

```html
<input
    value="India"
    readonly>
```

---

# disabled

## Purpose

Input becomes unusable.

Example:

```html
<input disabled>
```

Difference:

| Feature | readonly | disabled |
|-----------|----------|-----------|
| Editable | No | No |
| Submitted | Yes | No |
| Focusable | Yes | No |

---

# name Attribute

## What is it?

Defines the key sent to server.

Example:

```html
<input
    name="username">
```

Input:

```
John
```

Sent as:

```
username=John
```

Without name:

The value is NOT submitted.

---

# Common Beginner Mistakes

❌ Forgetting `name`

```html
<input type="text">
```

Value won't be submitted.

---

❌ Using GET for passwords

```html
<form method="GET">
```

Password becomes visible in URL.

---

❌ Missing labels

Hurts accessibility.

---

❌ Using radio buttons without same name

Results in multiple selections.

---

❌ Forgetting `required`

Allows empty submissions.

---

# Interview Questions

## Why is the `name` attribute important?

Because only inputs with a name are included in submitted form data.

---

## Difference between GET and POST?

GET sends data through URL.

POST sends data inside request body.

---

## Difference between radio and checkbox?

Radio:

Only one selection.

Checkbox:

Multiple selections.

---

## Difference between readonly and disabled?

Readonly submits data.

Disabled does not.

---

## Can forms work without JavaScript?

Yes.

HTML forms existed before JavaScript.

The browser itself can submit form data directly to servers.

# Mental Model

Think of a form as a paper application form.

```
<form>
    The paper itself

<input>
    Blank fields

<label>
    Field names

name
    Question identifiers

submit
    Handing the form to the office

action
    Office address

method
    Delivery method
</form>
```

If you understand this analogy, HTML forms become extremely intuitive.
