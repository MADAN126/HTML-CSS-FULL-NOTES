# HTML Audio & Video — Understanding-First Notes

---

# What are HTML Audio and Video?

HTML provides built-in elements that allow webpages to play media files directly inside the browser.

These elements are:

- `<audio>` → Plays sound files.
- `<video>` → Plays video files.

Before HTML5, websites relied on external plugins such as Flash to play media.

HTML5 eliminated this dependency by introducing native multimedia support.

---

# Why Do Audio and Video Tags Exist?

Imagine a modern website without multimedia support.

Users could:

- Read text
- View images
- Click links

But they could NOT:

- Watch tutorials
- Listen to podcasts
- Play music
- Stream lectures
- Watch advertisements

Audio and video tags solve this problem.

They allow browsers to handle multimedia natively.

---

# Evolution of Multimedia on the Web

## Before HTML5

Websites depended on plugins.

Example:

```
Website
    ↓
Flash Plugin
    ↓
Audio/Video Playback
```

Problems:

- Plugins had security vulnerabilities.
- Users had to install software.
- Performance issues occurred.
- Mobile devices often lacked support.

---

## After HTML5

Playback became built into browsers.

```
Website
    ↓
Browser
    ↓
Audio/Video Playback
```

Benefits:

- Faster
- Safer
- Easier to use
- Mobile-friendly

# The `<audio>` Element

---

# What is `<audio>`?

The `<audio>` element embeds sound content into a webpage.

Examples:

- Songs
- Podcasts
- Voice recordings
- Sound effects
- Audiobooks

---

# Basic Syntax

```html
<audio>
    <source src="song.mp3">
</audio>
```

---

# Simplest Example

```html
<audio src="song.mp3"></audio>
```

However, users cannot control playback.

---

# The controls Attribute

## What is it?

Displays browser-provided controls.

Example:

```html
<audio
    src="song.mp3"
    controls>
</audio>
```

---

## Browser Displays

```
▶ Play

⏸ Pause

🔊 Volume

⏱ Progress Bar
```

---

## Why Use It?

Without controls:

Users cannot interact with audio.

---

# Internal Working

Browser:

```
Downloads media
↓
Creates audio player UI
↓
Decodes audio
↓
Streams sound to speakers
```

---

# Supported Audio Formats

| Format | Extension |
|----------|------------|
| MP3 | .mp3 |
| WAV | .wav |
| OGG | .ogg |

---

# Why Multiple Formats?

Not all browsers support every format equally.

Providing multiple formats improves compatibility.

---

# Using Multiple Sources

Example:

```html
<audio controls>

    <source
        src="song.mp3"
        type="audio/mpeg">

    <source
        src="song.ogg"
        type="audio/ogg">

</audio>
```

---

# Internal Working

Browser checks:

```
Can I play MP3?
    ↓ Yes
Play MP3

No?
    ↓
Try OGG

No?
    ↓
Show fallback text
```

---

# Fallback Content

Example:

```html
<audio controls>

    <source src="song.mp3">

    Your browser does not support audio.

</audio>
```

Used when media cannot be played.

---

# autoplay Attribute

---

## Purpose

Starts playback automatically.

Example:

```html
<audio
    src="song.mp3"
    autoplay>
</audio>
```

---

## Modern Reality

Most browsers block autoplay.

Reason:

Unexpected sound annoys users.

---

# loop Attribute

---

## Purpose

Repeats audio continuously.

Example:

```html
<audio
    src="song.mp3"
    controls
    loop>
</audio>
```

---

# muted Attribute

---

## Purpose

Starts audio in muted state.

Example:

```html
<audio
    controls
    muted>
</audio>
```

---

# preload Attribute

---

## Purpose

Hints how much audio should load initially.

Values:

---

### none

```html
preload="none"
```

Load nothing.

---

### metadata

```html
preload="metadata"
```

Load basic information.

Example:

- Duration
- File size

---

### auto

```html
preload="auto"
```

Browser decides how much to load.

---

# Audio Example

```html
<audio
    controls
    loop>

    <source
        src="podcast.mp3"
        type="audio/mpeg">

    Your browser does not support audio.

</audio>
```

# The `<video>` Element

---

# What is `<video>`?

The `<video>` element embeds video content.

Examples:

- Tutorials
- Movies
- Product demos
- Advertisements
- Online courses

---

# Basic Syntax

```html
<video>
    <source src="video.mp4">
</video>
```

---

# Simplest Example

```html
<video
    src="movie.mp4"
    controls>
</video>
```

---

# Supported Video Formats

| Format | Extension |
|----------|------------|
| MP4 | .mp4 |
| WebM | .webm |
| OGG | .ogv |

---

# Multiple Sources

Example:

```html
<video controls>

    <source
        src="movie.mp4"
        type="video/mp4">

    <source
        src="movie.webm"
        type="video/webm">

</video>
```

---

# Internal Working

Browser checks:

```
Supports MP4?
    ↓ Yes
Play MP4

No?
    ↓
Try WebM

No?
    ↓
Show fallback text
```

---

# Width and Height

---

## Purpose

Control video dimensions.

Example:

```html
<video
    width="600"
    height="400"
    controls>

    <source src="movie.mp4">

</video>
```

---

# Poster Attribute

---

## What is it?

Displays an image before playback begins.

Example:

```html
<video
    controls
    poster="thumbnail.jpg">

    <source src="movie.mp4">

</video>
```

---

## Real Usage

YouTube previews work similarly.

---

# autoplay for Video

Example:

```html
<video
    autoplay
    muted
    controls>

    <source src="movie.mp4">

</video>
```

---

## Why muted?

Most browsers allow autoplay only if muted.

---

# Loop Video

Example:

```html
<video
    controls
    loop>

    <source src="movie.mp4">

</video>
```

---

# Video Example

```html
<video
    width="700"
    controls
    poster="preview.jpg">

    <source
        src="tutorial.mp4"
        type="video/mp4">

    Your browser does not support video.

</video>
```

# Subtitles and Captions

---

# The `<track>` Element

---

## What is it?

Provides subtitles, captions, or translations.

---

## Syntax

```html
<track>
```

---

## Example

```html
<video controls>

    <source src="movie.mp4">

    <track
        src="subtitles.vtt"
        kind="subtitles"
        srclang="en"
        label="English">

</video>
```

---

# Benefits

Improves:

- Accessibility
- Translation support
- User experience

---

# Audio vs Video

| Feature | Audio | Video |
|-----------|--------|--------|
| Tag | `<audio>` | `<video>` |
| Displays Visuals | No | Yes |
| Sound Support | Yes | Yes |
| Width/Height | Not Needed | Common |
| Poster Image | No | Yes |
| Captions | Rare | Common |

# Common Beginner Mistakes

❌ Forgetting `controls`

```html
<audio src="song.mp3"></audio>
```

User cannot play media.

---

❌ Relying only on autoplay.

Modern browsers often block it.

---

❌ Providing only one format.

Some browsers may fail to play it.

---

❌ Forgetting fallback text.

Older browsers need alternatives.

---

❌ Using huge video files.

Causes slow loading.

Compress videos appropriately.
