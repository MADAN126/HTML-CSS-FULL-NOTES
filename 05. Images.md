# HTML Images (`<img>` Tag)

---

# Why Do Images Exist?

Imagine a website that contains only text:

```text
Welcome to our restaurant.

We serve delicious food.

Our chefs are experts.
```

Now compare it with:

```text
[Photo of the Restaurant]

Welcome to our restaurant.

[Photo of Signature Dish]

We serve delicious food.

[Photo of Chef]

Our chefs are experts.
```

Which one is easier to understand and more engaging?

> Humans process visuals much faster than text.

Images exist to:

- Improve communication
- Increase engagement
- Provide visual context
- Enhance user experience
- Support branding
- Explain concepts quickly

Without images, websites would feel plain and less informative.

---

# What is an HTML Image?

An HTML image allows the browser to display visual content on a webpage.

HTML uses the `<img>` tag to embed images.

Example:

```html
<img src="cat.jpg" alt="A cute cat">
```

---

# Important Concept

The browser does NOT store the image inside HTML.

Instead:

```text
HTML
↓
Contains image address
↓
Browser requests image file
↓
Image is downloaded
↓
Image is displayed
```

---

# The `<img>` Tag

## Syntax

```html
<img src="image.jpg" alt="Description">
```

Example:

```html
<img src="dog.jpg" alt="Golden Retriever">
```

---

# Anatomy of an Image Tag

Example:

```html
<img src="cat.jpg" alt="Cute Cat">
```

Structure:

```text
<img
│
├── src="cat.jpg"
│       │
│       └─ Image Location
│
└── alt="Cute Cat"
        │
        └─ Alternative Description
>
```

---

# Why is `<img>` Special?

Unlike many HTML elements:

```html
<p>Text</p>
<div></div>
<a></a>
```

The image tag has NO closing tag.

Correct:

```html
<img src="cat.jpg" alt="Cat">
```

Wrong:

```html
<img src="cat.jpg"></img>
```

---

# What is `src`?

`src` means:

```text
Source
```

It tells the browser:

> "Where can I find this image?"

Without `src`, the browser cannot load an image.

Wrong:

```html
<img alt="Cat">
```

Correct:

```html
<img src="cat.jpg" alt="Cat">
```

---

# Types of Image Paths

There are two major types.

---

# 1. Relative Path

Used for images inside your project.

Project:

```text
website/
│
├── index.html
├── cat.jpg
└── images/
    └── dog.jpg
```

---

## Same Folder

```html
<img src="cat.jpg" alt="Cat">
```

Browser looks here:

```text
website/cat.jpg
```

---

## Inside Folder

```html
<img src="images/dog.jpg"
     alt="Dog">
```

Browser looks here:

```text
website/images/dog.jpg
```

---

## Move Up Folder

Example:

```html
<img src="../logo.png"
     alt="Logo">
```

```text
..
↓
Go back one folder
```

---

# 2. Absolute Path

Used for external images.

Example:

```html
<img src="https://example.com/cat.jpg"
     alt="Cat">
```

Browser:

```text
Internet
↓
example.com
↓
Download image
```

---

# What is `alt`?

`alt` means:

```text
Alternative Text
```

It describes the image.

Example:

```html
<img src="cat.jpg"
     alt="White cat sleeping">
```

---

# Why Does `alt` Exist?

It solves three problems.

---

## Problem 1: Broken Images

If image fails:

```text
Image Missing
↓
alt text shown instead
```

Example:

```html
<img src="missing.jpg"
     alt="Company Logo">
```

Output:

```text
Company Logo
```

---

## Problem 2: Accessibility

Screen readers read alt text.

Without alt:

```text
Image
Image
Image
```

With alt:

```text
Image:
Woman coding on laptop

Image:
Company logo

Image:
Chocolate cake
```

---

## Problem 3: SEO

Search engines cannot truly "see" images.

They rely partly on:

```text
alt text
```

to understand image content.

---

# Good vs Bad Alt Text

Good:

```html
alt="Golden Retriever running in park"
```

Bad:

```html
alt="dog"
```

Very Bad:

```html
alt="image123"
```

---

# Decorative Images

If image has NO meaning:

Example:

```text
Background flourish
Visual divider
Decoration
```

Use:

```html
alt=""
```

This tells screen readers:

> Ignore this image.

---

# Width and Height

Control image size.

Example:

```html
<img src="cat.jpg"
     alt="Cat"
     width="300"
     height="200">
```

---

# Internal Browser Behavior

Without dimensions:

```text
Page loads
↓
Image size unknown
↓
Layout shifts
```

With dimensions:

```text
Page loads
↓
Space reserved
↓
Stable layout
```

---

# Which Unit?

HTML attributes use pixels.

Example:

```html
width="300"
height="200"
```

means:

```text
300px × 200px
```

# Image Formats

## JPEG (.jpg)

Best for:

```text
Photographs
Real-world images
```

Advantages:

- Small size
- Millions of colors

Disadvantages:

- Lossy compression

---

## PNG (.png)

Best for:

```text
Logos
Icons
Transparent images
```

Advantages:

- Transparency
- High quality

Disadvantages:

- Larger files

---

## GIF (.gif)

Best for:

```text
Simple animations
```

Advantages:

- Animation support

Disadvantages:

- Limited colors

---

## SVG (.svg)

Best for:

```text
Icons
Logos
Illustrations
```

Advantages:

- Infinite scaling
- Small size

Disadvantages:

- Not suitable for photographs

---

## WebP (.webp)

Modern image format.

Advantages:

- Smaller files
- High quality
- Transparency

Recommended for modern websites.

---

# Clickable Images

Images can act as links.

Example:

```html
<a href="https://github.com">
    <img src="logo.png"
         alt="GitHub Logo">
</a>
```

Flow:

```text
User Clicks Image
↓
Link Activated
↓
Navigate
```

---

# Figure and Figcaption

Used when image needs explanation.

Example:

```html
<figure>
    <img src="mountain.jpg"
         alt="Snow-covered mountain">

    <figcaption>
        Mount Everest during sunrise.
    </figcaption>
</figure>
```

---

# Why Use Figure?

Because browser understands:

```text
Image
↓
Caption belongs to image
```

Semantic meaning improves accessibility.

---

# Browser Workflow

When browser sees:

```html
<img src="cat.jpg"
     alt="Cat">
```

It does:

```text
Parse HTML
↓
Find src
↓
Request image
↓
Download image
↓
Render image
↓
Use alt if image fails
```

---

# Common Mistakes

## Mistake 1

Missing alt text.

Wrong:

```html
<img src="dog.jpg">
```

Correct:

```html
<img src="dog.jpg"
     alt="Golden Retriever">
```

---

## Mistake 2

Huge image files.

Wrong:

```text
10MB image
```

Result:

```text
Slow website
```

Better:

```text
Compressed image
WebP
Optimized JPEG
```

---

## Mistake 3

Using PNG for photographs.

Wrong:

```text
Vacation photo → PNG
```

Better:

```text
Vacation photo → JPEG/WebP
```

---

## Mistake 4

Using meaningless alt text.

Wrong:

```html
alt="picture"
```

Better:

```html
alt="Student reading a programming book"
```

```text
<img>
```

```text
src

Where is the photo?

alt

What should people know if they cannot see it?

How much space should it occupy?
```
