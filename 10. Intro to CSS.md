# CSS Introduction

---

# What is CSS?

**CSS (Cascading Style Sheets)** is the language used to style and design HTML documents.

HTML defines the structure and content of a webpage.

CSS controls how that content looks.

Think of it like this:

| Technology | Responsibility |
|---|---|
| HTML | What exists on the page |
| CSS | How it looks |
| JavaScript | How it behaves |

Example:

Without CSS:

```html
<h1>Hello World</h1>
```

Browser output:

Plain black text.

With CSS:

```html
<h1>Hello World</h1>

<style>
h1{
    color:red;
    font-size:50px;
}
</style>
```

Output:

Large red heading.

---

# Why Does CSS Exist?

Before CSS, HTML was used for both structure and presentation.

Example:

```html
<font color="red" size="7">
Welcome
</font>
```

Problems:

- HTML became messy.
- Same styling repeated everywhere.
- Difficult maintenance.
- Poor separation of concerns.

CSS solved this by separating:

- Structure → HTML
- Presentation → CSS

Step 1:

Browser reads HTML.

Example:

```html

```

↓

Step 2:

Browser builds DOM.

```text
Document
 └─ h1
```

↓

Step 3:

Browser reads CSS.

```css
h1{
    color:red;
}
```

↓

Step 4:

Matches selector.

```text
Does h1 exist?

Yes.
```

↓

Step 5:

Applies styles.

↓

Step 6:

Creates Render Tree.

↓

Step 7:

Paints webpage.

# Ways to Add CSS

---

## 1. Inline CSS

Inside HTML tag.

```html
<h1 style="color:red;">
Hello
</h1>
```

### Advantages

- Quick testing.

### Disadvantages

- Hard to maintain.
- Repetition.
- Not reusable.

### Avoid

Large projects.

---

## 2. Internal CSS

Inside:

```html
<style>
</style>
```

Example:

```html
<head>
    <style>
        h1{
            color:red;
        }
    </style>
</head>
```

### Advantages

- Better than inline.

### Disadvantages

- Only works for one page.

### Use

Small pages.

---

## 3. External CSS

Separate file.

styles.css

```css
h1{
    color:red;
}
```

HTML:

```html
<link rel="stylesheet" href="styles.css">
```

### Advantages

- Reusable.
- Professional.
- Easy maintenance.
- Faster loading through caching.

### Use

Real projects.

# CSS Selectors

Selectors determine WHAT gets styled.

---

## Element Selector

Targets tags.

```css
p{
    color:blue;
}
```

Targets:

```html
<p>One</p>
<p>Two</p>
```

---

## ID Selector

Uses:

```css
#header{
    color:red;
}
```

HTML:

```html
<h1 id="header">
Welcome
</h1>
```

### Rule

ID should be unique.

---

## Class Selector

Uses:

```css
.note{
    color:green;
}
```

HTML:

```html
<p class="note">A</p>
<p class="note">B</p>
```

Reusable.

---

## Universal Selector

Targets everything.

```css
*{
    margin:0;
}
```

Often used for CSS reset.

---

## Group Selector

```css
h1,p,div{
    color:blue;
}
```

Styles multiple elements.

---

# Cascade

CSS = Cascading Style Sheets.

"Cascading" means:

When multiple styles apply, browser decides which wins.

Example:

```css
p{
    color:red;
}

p{
    color:blue;
}
```

Result:

```text
Blue
```

Because later rule overrides earlier rule.

---

# Specificity

Determines priority.

Priority:

```text
Inline CSS
    ↓
ID
    ↓
Class
    ↓
Element
```

Example:

```css
p{
    color:red;
}

.note{
    color:green;
}

#intro{
    color:blue;
}
```

HTML:

```html
<p id="intro" class="note">
Hello
</p>
```

Output:

```text
Blue
```

ID wins.

---

# Inheritance

Some properties automatically pass from parent to child.

Example:

```html
<body>
    <div>
        <p>Hello</p>
    </div>
</body>
```

CSS:

```css
body{
    color:red;
}
```

Output:

```text
Hello becomes red.
```

Inherited properties:

- color
- font-family
- text-align

Not inherited:

- margin
- padding
- border

# Box Model (Most Important Concept)

Everything in CSS is a box.

Structure:

```text
+---------------------+
|       Margin        |
|  +---------------+  |
|  |    Border     |  |
|  | +-----------+ |  |
|  | | Padding   | |  |
|  | | Content   | |  |
|  | +-----------+ |  |
|  +---------------+  |
+---------------------+
```

---

---

## Border

---

## Margin

```css

```

# Common Beginner Mistakes

## Forgetting semicolon

Wrong:

```css
color:red
font-size:20px;
```

Correct:

```css
color:red;
font-size:20px;
```

---

## Using IDs everywhere

Bad:

```css
#box1
#box2
#box3
```

Better:

```css
.card
.card
.card
```

Use classes for reusable styles.

---

## Using Inline CSS excessively

Bad:

```html
<p style="color:red">
```

Use external CSS.

---

## Ignoring Specificity

Thinking:

```text
Last rule always wins.
```

Not true.

Specificity matters.

# Interview Questions

## What is CSS?

Language used to style HTML documents.

---

## Why is CSS called Cascading?

Because multiple rules cascade together and the browser decides which one applies based on specificity and order.

---

## Difference Between ID and Class?

| ID | Class |
|---|---|
| Unique | Reusable |
| # | . |
| Higher specificity | Lower specificity |
| One element | Multiple elements |

---

## Which CSS type is preferred?

```text
External CSS
```

Because it is reusable and maintainable.

---

Every HTML element is treated as:

```text
Content
↓
Padding
↓
Border
↓
Margin
```
