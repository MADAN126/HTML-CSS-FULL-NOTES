# HTML Links (`<a>` Tag)

---

# Why Do Links Exist?

The web is called the **World Wide Web** because pages are connected together through links.

Without links:

```text
Page A

Page B

Page C
```

These pages would exist in isolation.

With links:

```text
Page A ──► Page B ──► Page C
       └──────────► Page D
```

they become a network of connected information.

> Links are the foundation of the web.

---

# What is a Link?

A link allows users to navigate from one resource to another.

Resources can be:

- Web pages
- Images
- PDFs
- Videos
- Email addresses
- Phone numbers
- Specific sections of a page
- Downloadable files

HTML uses the **anchor tag** to create links.

```html
<a>...</a>
```

---

# The Anchor Tag

## Syntax

```html
<a href="URL">Link Text</a>
```

Example:

```html
<a href="https://www.google.com">
    Visit Google
</a>
```

Output:

```text
Visit Google
```

(clickable)

---

# Anatomy of a Link

Example:

```html
<a href="https://www.google.com">
    Visit Google
</a>
```

Structure:

```text
<a
│
├── href="https://www.google.com"
│       │
│       └─ Destination
│
└── Visit Google
        │
        └─ Visible clickable text
```

---

# What is `href`?

`href`

stands for:

```text
Hypertext Reference
```

It tells the browser:

> "Where should I go when this link is clicked?"

Without `href`, the anchor is not a real link.

Wrong:

```html
<a>Google</a>
```

Correct:

```html
<a href="https://www.google.com">
    Google
</a>
```

---

# Absolute URLs

## What are they?

Complete web addresses.

Example:

```html
<a href="https://www.wikipedia.org">
    Wikipedia
</a>
```

---

## Structure

```text
https://www.wikipedia.org
│
├─ Protocol
│
└─ Domain
```

---

## Use When

Use for external websites.

Examples:

```html
<a href="https://github.com">GitHub</a>

<a href="https://www.youtube.com">YouTube</a>

<a href="https://openai.com">OpenAI</a>
```

---

# Relative URLs

## What are they?

Links pointing to files within your own website.

Example folder:

```text
project/
│
├── index.html
├── about.html
└── contact.html
```

---

## Example

Inside index.html:

```html
<a href="about.html">
    About Us
</a>
```

---

## How Browser Resolves It

Current page:

```text
project/index.html
```

Link:

```text
about.html
```

Browser interprets:

```text
project/about.html
```

---

# Relative URL Examples

## Same Folder

```html
<a href="about.html">
    About
</a>
```

---

## Inside Folder

```html
<a href="pages/about.html">
    About
</a>
```

---

## Move Up One Folder

```html
<a href="../index.html">
    Home
</a>
```

---

## Move Up Two Folders

```html
<a href="../../index.html">
    Home
</a>
```

# Opening Links in New Tabs

## `target` Attribute

Controls where the linked page opens.

---

## Same Tab (Default)

```html
<a href="https://github.com">
    GitHub
</a>
```

---

## New Tab

```html
<a href="https://github.com"
   target="_blank">
    GitHub
</a>
```

---

# Common Target Values

| Value | Meaning |
|---------|---------|
| `_self` | Current tab |
| `_blank` | New tab |
| `_parent` | Parent frame |
| `_top` | Full window |

---

# Why Use `_blank`?

Useful for external websites.

Example:

```html
<a href="https://youtube.com"
   target="_blank">
    Open YouTube
</a>
```

Your website remains open.

---

# Security with `_blank`

Wrong:

```html
<a href="https://example.com"
   target="_blank">
    Open
</a>
```

Better:

```html
<a href="https://example.com"
   target="_blank"
   rel="noopener noreferrer">
    Open
</a>
```

---

# Why `rel="noopener noreferrer"`?

Prevents the new page from accessing your original page.

Improves:

- Security
- Performance

Common interview question.

# Email Links

Links can open the user's email application.

Syntax:

```html
<a href="mailto:john@example.com">
    Email John
</a>
```

---

# Example

```html
<a href="mailto:support@example.com">
    Contact Support
</a>
```

Clicking opens:

```text
Outlook
Gmail
Apple Mail
etc.
```

---

# Email with Subject

Example:

```html
<a href="mailto:support@example.com?subject=Help">
    Email Support
</a>
```

---

# Phone Links

Useful for mobile devices.

Syntax:

```html
<a href="tel:+911234567890">
    Call Us
</a>
```

---

# Example

```html
<a href="tel:+919876543210">
    Contact Sales
</a>
```

Mobile browser opens dialer.

---

# Download Links

Force download of a file.

Example:

```html
<a href="resume.pdf" download>
    Download Resume
</a>
```

---

# Custom Download Filename

Example:

```html
<a href="resume.pdf"
   download="MyResume.pdf">
    Download Resume
</a>
```

---

# Linking to Images

Images can become clickable.

Example:

```html
<a href="https://github.com">
    <img src="logo.png"
         alt="GitHub Logo">
</a>
```

Clicking the image opens GitHub.

---

# Bookmark Links (Page Sections)

Jump to another section of the same page.

---

## Step 1

Add an ID.

```html
<h2 id="contact">
    Contact Us
</h2>
```

---

## Step 2

Create link.

```html
<a href="#contact">
    Go to Contact
</a>
```

---

# Internal Flow

```text
Click Link
     ↓
href="#contact"
     ↓
Find id="contact"
     ↓
Scroll to section
```

---

# Linking Between Pages and Sections

Example:

```html
<a href="about.html#team">
    Meet Our Team
</a>
```

Browser:

```text
Open about.html
↓
Find id="team"
↓
Scroll there
```

# Link States

Browsers track link states.

---

## Unvisited

```css
a:link
```

Never clicked.

---

## Visited

```css
a:visited
```

Already clicked.

---

## Hover

```css
a:hover
```

Mouse over link.

---

## Active

```css
a:active
```

Being clicked.

# Browser Workflow

When user clicks:

```text
User Clicks
      ↓
Browser Reads href
      ↓
Determines Destination
      ↓
Loads Resource
      ↓
Displays New Content
```

# Accessibility Best Practices

Good:

```html
<a href="guide.pdf">
    Download HTML Guide
</a>
```

---

Bad:

```html
<a href="guide.pdf">
    Click Here
</a>
```

---

# Why?

Screen readers announce links.

Users hear:

Bad:

```text
Click here
Click here
Click here
```

Good:

```text
Download HTML Guide
View Pricing
Contact Support
```

# Common Mistakes

## Mistake 1

Missing `href`

Wrong:

```html
<a>Google</a>
```

Correct:

```html
<a href="https://google.com">
    Google
</a>
```

---

## Mistake 2

Using absolute URLs internally.

Wrong:

```html
<a href="https://mysite.com/about.html">
```

Better:

```html
<a href="about.html">
```

---

## Mistake 3

Using `_blank` without `rel`

Wrong:

```html
<a target="_blank">
```

Better:

```html
<a target="_blank"
   rel="noopener noreferrer">
```

---

## Mistake 4

Using vague text.

Wrong:

```html
Click Here
```

Better:

```html
Read the HTML Documentation
```
