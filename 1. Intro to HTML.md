# HTML Introduction

## What is HTML?

HTML stands for **HyperText Markup Language**.

It is the standard markup language used to create the structure of web pages.

HTML is not a programming language because it does not contain logic such as loops, conditions, or variables. Instead, it describes **what content exists on a web page** and how that content is organized.

Examples of content HTML can describe:

- Headings
- Paragraphs
- Images
- Links
- Tables
- Forms
- Videos
- Navigation menus

---

## Why Does HTML Exist?

Before HTML, there was no standard way to share structured documents across different computers connected through networks.

HTML was introduced to solve this problem.

It provides a universal language that browsers understand.

### Problems HTML Solves

- Defines the structure of web documents.
- Allows documents to be linked together using hyperlinks.
- Enables browsers to display content consistently.
- Separates content from presentation and behavior.

Without HTML:

- Browsers would not know what content means.
- Websites could not have standardized structures.
- The Web as we know it would not exist.

## What Does HTML Stand For?

### HyperText

Text connected using links.

Example:

You click a link on one page and move to another page.

That connection is called HyperText.

### Markup

Special tags used to describe content.

Example:

```html
<h1>Welcome</h1>
```

The browser understands that this is a heading.

### Language

A set of rules used to describe documents.

---

## How HTML Works Internally

Execution Flow:

```
User requests website
        ↓
Browser receives HTML file
        ↓
Browser reads HTML top to bottom
        ↓
Browser creates DOM Tree
        ↓
CSS styles the DOM
        ↓
JavaScript adds behavior
        ↓
Final webpage displayed
```

---

## HTML Document Structure

Basic HTML document:

```html
<!DOCTYPE html>
<html>
<head>
    <title>My First Page</title>
</head>

<body>
    <h1>Hello World</h1>
    <p>Welcome to HTML.</p>
</body>
</html>
```

---

## Understanding Each Part

### 1. `<!DOCTYPE html>`

#### What is it?

Declares that the document uses HTML5.

#### Why does it exist?

It tells the browser which version of HTML rules to follow.

#### Without it

The browser may enter quirks mode and render pages differently.

---

### 2. `<html>`

#### What is it?

Root element of the webpage.

#### Purpose

Everything inside the page lives within this tag.

Example:

```html
<html>
    ...
</html>
```

---

### 3. `<head>`

#### What is it?

Contains information about the page.

#### Not visible to users.

Contains:

- Title
- Meta information
- CSS links
- JavaScript links

Example:

```html
<head>
    <title>My Website</title>
</head>
```

---

### 4. `<title>`

#### Purpose

Defines the text shown in:

- Browser tabs
- Search engine titles

Example:

```html
<title>Amazon</title>
```

Output:

Browser tab displays:

```
Amazon
```

---

### 5. `<body>`

#### Purpose

Contains everything visible on the webpage.

Examples:

- Text
- Images
- Buttons
- Tables
- Forms

Example:

```html
<body>
    <h1>Welcome</h1>
</body>
```

---

# HTML Elements

An HTML element consists of:

```html
<tagname>Content</tagname>
```

Example:

```html
<p>Hello</p>
```

Element Breakdown:

| Part | Meaning |
|---------|---------|
| `<p>` | Opening tag |
| Hello | Content |
| `</p>` | Closing tag |

---

## Empty Elements

Some elements do not require closing tags.

Example:

```html
<br>
<hr>
<img>
<input>
```

These are called void elements.

---

Examples:

| Tag | Purpose |
|-------|---------|
| `<h1>` | Largest heading |
| `<p>` | Paragraph |
| `<a>` | Link |
| `<img>` | Image |
| `<ul>` | Unordered list |
| `<ol>` | Ordered list |
| `<table>` | Table |

# HTML Attributes

Attributes provide extra information.

Syntax:

```html
<tag attribute="value">
```

Example:

```html
<a href="https://google.com">
    Google
</a>
```

Here:

- href = attribute
- URL = value

---

## Common Attributes

| Attribute | Purpose |
|-----------|----------|
| id | Unique identifier |
| class | Group elements |
| href | Link destination |
| src | Resource path |
| alt | Alternative image text |
| style | Inline styling |
| title | Tooltip text |

# Headings

HTML provides six heading levels.

Example:

```html
<h1>Main Heading</h1>
<h2>Sub Heading</h2>
<h3>Section</h3>
<h4>Subsection</h4>
<h5>Small Heading</h5>
<h6>Smallest Heading</h6>
```

Use headings hierarchically.

Do not use them merely to increase font size.

---

# Paragraphs

Used to display text blocks.

Example:

```html
<p>
HTML is used to structure web pages.
</p>
```

Output:

A paragraph of text appears.

---

# Links

Links connect web pages.

Syntax:

```html
<a href="https://google.com">
    Visit Google
</a>
```

Output:

A clickable hyperlink.

---

# Images

Display images.

Syntax:

```html
<img src="cat.jpg" alt="Cat">
```

Attributes:

- src → image location
- alt → description

Why alt matters:

- Accessibility
- SEO
- Fallback text

# Deep Internal Understanding

When a browser receives HTML:

```
HTML File
    ↓
Tokenizer
    ↓
Parser
    ↓
DOM Nodes Created
    ↓
DOM Tree Built
    ↓
CSS Applied
    ↓
Layout Calculated
    ↓
Painting
    ↓
Webpage Displayed
```

DOM = Document Object Model.

The DOM is a tree representation of HTML.

Example:

```html
<body>
    <h1>Hello</h1>
    <p>World</p>
</body>
```

DOM:

```
body
├── h1
│   └── Hello
└── p
    └── World
```

# Common Beginner Mistakes

## Forgetting Closing Tags

Wrong:

```html
<p>Hello
```

Correct:

```html
<p>Hello</p>
```

---

## Using Headings for Styling

Wrong:

```html
<h1>Big Text</h1>
```

Only because it looks bigger.

Correct:

Use headings based on meaning.

Use CSS for size.

---

## Ignoring alt Attributes

Wrong:

```html
<img src="dog.jpg">
```

Correct:

```html
<img src="dog.jpg" alt="Brown dog">
```

---

# Best Practices

- Use semantic HTML.
- Indent nested elements.
- Use meaningful headings.
- Always provide alt text.
- Keep structure clean.
- Separate HTML, CSS, and JavaScript.
- Validate HTML regularly.

# Things Beginners Usually Misunderstand

### "HTML builds websites completely."

Incorrect.

HTML only structures the page.

CSS styles it.

JavaScript adds behavior.

---

### "Tags control appearance."

Incorrect.

Tags describe meaning.

CSS controls appearance.

---

### "HTML is a programming language."

Incorrect.

It is a markup language.
