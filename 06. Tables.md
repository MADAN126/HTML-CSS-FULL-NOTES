# HTML Tables

---

# Why Do Tables Exist?

Imagine displaying employee information like this:

```text
John   25   Developer
Sarah  28   Designer
Mike   30   Tester
```

It quickly becomes difficult to read.

Tables exist to organize information into:

```text
Rows
↓
Columns
↓
Cells
```

making relationships between data clear.

Example:

```text
+---------+-----+------------+
| Name    | Age | Role       |
+---------+-----+------------+
| John    | 25  | Developer  |
| Sarah   | 28  | Designer   |
| Mike    | 30  | Tester     |
+---------+-----+------------+
```

> Tables solve the problem of presenting structured data.

---

# What is an HTML Table?

An HTML table is used to display **tabular data**.

Tabular data means:

```text
Information arranged in rows and columns.
```

Examples:

- Student marks
- Employee records
- Product pricing
- Timetables
- Reports
- Financial statements

---

# Important Principle

Tables are for:

```text
DATA
```

NOT for:

```text
PAGE LAYOUT
```

Old websites used tables for layouts.

Modern websites use:

```text
CSS Flexbox
CSS Grid
```

for layouts.

---

# Basic Table Structure

```html
<table>
    <tr>
        <td>John</td>
        <td>25</td>
    </tr>

    <tr>
        <td>Sarah</td>
        <td>28</td>
    </tr>
</table>
```

---

# Output

```text
+-------+----+
| John  | 25 |
+-------+----+
| Sarah | 28 |
+-------+----+
```

---

# Anatomy of a Table

```html
<table>
    <tr>
        <td>A</td>
        <td>B</td>
    </tr>
</table>
```

Structure:

```text
table
│
└── tr
    │
    ├── td
    │
    └── td
```

---

# Table Tags Overview

| Tag | Meaning |
|-------|----------|
| `<table>` | Entire table |
| `<tr>` | Table row |
| `<td>` | Table data cell |
| `<th>` | Table header cell |
| `<thead>` | Header section |
| `<tbody>` | Body section |
| `<tfoot>` | Footer section |
| `<caption>` | Table title |

---

# `<table>` Tag

## What is it?

Container for the entire table.

Example:

```html
<table>
</table>
```

Think of it as:

```text
The outer box.
```

---

# `<tr>` Tag

## What is it?

Represents a table row.

Example:

```html
<tr>
</tr>
```

Think of it as:

```text
Horizontal line of cells.
```

---

# `<td>` Tag

## What is it?

Represents actual data.

Example:

```html
<td>John</td>
```

Output:

```text
John
```

---

# Example

```html
<table>
    <tr>
        <td>John</td>
        <td>Developer</td>
    </tr>
</table>
```

Output:

```text
John | Developer
```

---

# `<th>` Tag

## What is it?

Defines header cells.

Example:

```html
<th>Name</th>
```

Browsers display it as:

```text
Bold
Centered
```

by default.

---

# Example

```html
<table>
    <tr>
        <th>Name</th>
        <th>Age</th>
    </tr>

    <tr>
        <td>John</td>
        <td>25</td>
    </tr>
</table>
```

Output:

```text
+-------+-----+
| Name  | Age |
+-------+-----+
| John  | 25  |
+-------+-----+
```

---

# Why Use `<th>`?

Because browsers and screen readers understand:

```text
These cells are labels.
```

Accessibility improves.

---

# Complete Example

```html
<table>
    <tr>
        <th>Name</th>
        <th>Age</th>
        <th>Role</th>
    </tr>

    <tr>
        <td>John</td>
        <td>25</td>
        <td>Developer</td>
    </tr>

    <tr>
        <td>Sarah</td>
        <td>28</td>
        <td>Designer</td>
    </tr>
</table>
```

Output:

```text
+---------+-----+------------+
| Name    | Age | Role       |
+---------+-----+------------+
| John    | 25  | Developer  |
| Sarah   | 28  | Designer   |
+---------+-----+------------+
```

# Table Sections

Large tables can be divided into sections.

```text
Header
Body
Footer
```

---

# `<thead>`

Contains headings.

Example:

```html
<thead>
    <tr>
        <th>Name</th>
        <th>Age</th>
    </tr>
</thead>
```

---

# `<tbody>`

Contains actual records.

Example:

```html
<tbody>
    <tr>
        <td>John</td>
        <td>25</td>
    </tr>
</tbody>
```

---

# `<tfoot>`

Contains summaries.

Example:

```html
<tfoot>
    <tr>
        <td>Total</td>
        <td>100</td>
    </tr>
</tfoot>
```

---

# Complete Semantic Table

```html
<table>

    <caption>Employee Details</caption>

    <thead>
        <tr>
            <th>Name</th>
            <th>Age</th>
            <th>Role</th>
        </tr>
    </thead>

    <tbody>
        <tr>
            <td>John</td>
            <td>25</td>
            <td>Developer</td>
        </tr>

        <tr>
            <td>Sarah</td>
            <td>28</td>
            <td>Designer</td>
        </tr>
    </tbody>

    <tfoot>
        <tr>
            <td colspan="3">
                Total Employees: 2
            </td>
        </tr>
    </tfoot>

</table>
```

---

# `<caption>`

Provides a title.

Example:

```html
<caption>
    Student Marks
</caption>
```

Output:

```text
Student Marks
+---------+
| Table   |
+---------+
```

---

# Why Use Caption?

It tells users:

```text
What is this table about?
```

Improves accessibility.

---

# `colspan`

Merge columns.

Example:

```html
<td colspan="2">
    Combined Cell
</td>
```

---

# Example

```html
<table>

<tr>
    <td colspan="2">
        Total
    </td>
</tr>

</table>
```

Output:

```text
+-------------+
|    Total    |
+-------------+
```

---

# Why Use `colspan`?

Useful for:

- Totals
- Group headings
- Reports

---

# `rowspan`

Merge rows.

Example:

```html
<td rowspan="2">
    HR
</td>
```

---

# Example

```html
<table>

<tr>
    <td rowspan="2">HR</td>
    <td>John</td>
</tr>

<tr>
    <td>Sarah</td>
</tr>

</table>
```

Output:

```text
+------+--------+
| HR   | John   |
|      +--------+
|      | Sarah  |
+------+--------+
```

---

# Internal Browser Understanding

Example:

```html
<table>

<tr>
    <th>Name</th>
</tr>

<tr>
    <td>John</td>
</tr>

</table>
```

DOM Tree:

```text
Document
│
└── table
    │
    ├── tr
    │   └── th
    │
    └── tr
        └── td
```

Browser understands relationships.

# Accessibility

Screen readers announce:

```text
Table with 3 columns and 2 rows.

Column:
Name

Value:
John
```

instead of:

```text
John 25 Developer
```

# Common Mistakes

## Mistake 1

Using tables for layout.

Wrong:

```text
Entire website built with tables.
```

Use:

```text
Flexbox
Grid
```

instead.

---

## Mistake 2

Using `<td>` for headers.

Wrong:

```html
<td>Name</td>
```

Correct:

```html
<th>Name</th>
```

---

## Mistake 3

Skipping `<caption>`.

Large tables become confusing.

---

## Mistake 4

Overusing `rowspan` and `colspan`.

Can make tables difficult to understand.

# Interview Questions

## Which tag creates a table?

Answer:

```text
<table>
```

---

## Difference between `<td>` and `<th>`?

Answer:

```text
<td>
↓
Normal data cell

<th>
↓
Header cell
Semantic meaning
Bold by default
```

---

## What does `colspan` do?

Answer:

```text
Merges multiple columns into one cell.
```

---

## What does `rowspan` do?

Answer:

```text
Merges multiple rows into one cell.
```

---

## Should tables be used for layouts?

Answer:

```text
No.

Use Flexbox or Grid.

Tables are for tabular data only.
```

# Mental Model

Think of tables like spreadsheets.

```text
<table>
↓
Spreadsheet

<tr>
↓
Horizontal row

<td>
↓
Individual value

<th>
↓
Column label

<thead>
↓
Headings

<tbody>
↓
Actual records

<tfoot>
↓
Summary
```

Use tables when:

```text
Data has relationships across rows and columns.
```

Avoid tables when:

```text
Building page layouts.
```
