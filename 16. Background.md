# CSS Background

---

# What is a Background?

A background is the visual layer painted behind an element's content.

It determines:

- The color behind the content.
- The image behind the content.
- How that image behaves.
- Where that image is positioned.
- Whether the image repeats.
- How much area the background covers.

Think of it as:

> The wallpaper of an element.

---

# Why Do Backgrounds Exist?

Without backgrounds:

- Every webpage would look flat.
- Important sections would not stand out.
- Components would blend together.
- Visual hierarchy would be difficult to establish.

Backgrounds help us:

- Highlight content.
- Improve readability.
- Create attractive interfaces.
- Establish branding.
- Guide user attention.

# background-color

Defines the color behind an element.

Syntax:

```css
selector{
    background-color: value;
}
```

Example:

```css
body{
    background-color: lightblue;
}
```

---

# Why Use background-color?

It helps:

- Separate sections.
- Improve contrast.
- Match brand identity.
- Increase readability.

---

# background-image

Places an image behind the content.

Syntax:

```css
background-image: url("image.jpg");
```

Example:

```css
body{
    background-image: url("nature.jpg");
}
```

---

# Why Use Background Images?

Useful for:

- Hero sections
- Decorative patterns
- Branding
- Landing pages
- Promotional banners

# background-position

Determines where the image appears.

Syntax:

```css
background-position: value;
```

---

## Keywords

```css
background-position: center;
background-position: top;
background-position: bottom;
background-position: left;
background-position: right;
```

---

## Combined Keywords

```css
background-position: top left;
background-position: center center;
background-position: bottom right;
```

---

## Coordinates

```css
background-position: 50px 20px;
```

Means:

```
50px from left
20px from top
```

---

# background-size

Controls image size.

Syntax:

```css
background-size: value;
```

---

## Auto (Default)

```css
background-size: auto;
```

Uses original image dimensions.

---

## Specific Dimensions

```css
background-size: 300px 200px;
```

Width:

```
300px
```

Height:

```
200px
```

---

## cover

Extremely important.

```css
background-size: cover;
```

Behavior:

```
Fill entire container.
Maintain aspect ratio.
Crop overflow if necessary.
```

Example:

```
Container completely covered.
```

Used heavily in hero sections.

---

## contain

```css
background-size: contain;
```

Behavior:

```
Entire image visible.
Maintain aspect ratio.
May leave empty space.
```

Example:

```
Image fully visible.
Container may not be filled.
```

---

# cover vs contain

## cover

Goal:

> Fill container.

## contain

> Show entire image.

# background-attachment

Controls scrolling behavior.

Syntax:

```css
background-attachment: value;
```

---

## scroll (Default)

```css
background-attachment: scroll;
```

Background moves with content.

---

## fixed

```css
background-attachment: fixed;
```

Background stays fixed.

Content scrolls over it.

Creates:

```
Parallax-like effect
```

---

## local

```css
background-attachment: local;
```

Scrolls with element contents.

Rarely used.

---

# background-origin

Determines where the background positioning starts.

Syntax:

```css
background-origin: value;
```

---

## padding-box

Default.

Starts from padding area.

---

## border-box

Starts from border.

---

## content-box

Starts from content area.

---

# background-clip

Determines where the background is painted.

Syntax:

```css
background-clip: value;
```

---

## border-box

Paints into border.

---

## padding-box

Stops before border.

---

## content-box

Only paints content area.

---

# Background Shorthand

Instead of:

```css
background-color: black;
background-image: url("img.jpg");
background-repeat: no-repeat;
background-position: center;
background-size: cover;
```

Use:

```css
background:
black
url("img.jpg")
no-repeat
center/cover;
```

Modern CSS often uses shorthand.

# Multiple Backgrounds

CSS supports multiple backgrounds.

Example:

```css
background:
url("top.png"),
url("bottom.png"),
lightblue;
```

Layers:

```
Top Image
↓
Bottom Image
↓
Background Color
```

# Linear Gradients

A gradient is treated as a background image.

Example:

```css
background:
linear-gradient(red, blue);
```

Result:

```
Red
↓
Purple
↓
Blue
```

---

## Direction

```css
background:
linear-gradient(to right, red, blue);
```

Left:

```
Red
```

Right:

```
Blue
```

---

## Multiple Colors

```css
background:
linear-gradient(red, yellow, green);
```

Transitions smoothly.

---

# Radial Gradients

Start from center.

Example:

```css
background:
radial-gradient(red, blue);
```

Result:

```
Blue

Purple

Red
```

Expands outward.

# Common Mistakes

---

## Forgetting no-repeat

Wrong:

```css
background-image: url("logo.png");
```

Unexpected:

```
Logo repeated everywhere.
```

Correct:

```css
background-repeat: no-repeat;
```

---

## Using Fixed Sizes Everywhere

Bad:

```css
background-size: 500px 500px;
```

May break responsiveness.

Better:

```css
background-size: cover;
```

---

## Confusing cover and contain

Remember:

Cover:

> Fill container.

Contain:

> Show full image.

---

## Poor Contrast

Bad:

```css
background-color: black;
color: darkblue;
```

Hard to read.

Always ensure accessibility.


Whenever you think:

> "What should this element look like behind the content?"

you are thinking about the **background**.
